인터넷에서 컴퓨터 둘은 어떻게 통신할까?
-> IP(인터넷 프로토콜) 
(우리가 흔히 쓰는 100.100.100.1 은 IP의 주소체계를 의미)

IP(인터넷 프로토콜)
- 지정한 IP 주소에 데이터 전달
- 패킷 단위로 전달

패킷
- 출발지 IP, 목적지 IP, 메시지 포함

인터넷에 던져진 패킷은 인터넷상 노드를 따라 목적지 IP로 이동하게 된다.
메시지를 받게 된 곳에서는 다시 응답 패킷을 보낸다.

IP 프로토콜의 한계
- 비 연결성(Unreliability)
= 패킷을 받을 대상이 없거나 서비스 불능 상태여도 패킷 전송
- 비 신뢰성(Connectionlessness)
= 중간에 패킷이 사라질수도 있음
= 패킷이 순서대로 오지 않을수도 있음

즉 IP는 내용물이 무엇인지, 받는 상대는 어떤 상태인지 등을 확인하지 않고 전송 요청이 있으면
무조건 보낸다. 전송이 제대로 되었는지, 전송과정은 멀쩡한지등은 보장하지 않는다.

TCP(전송 제어 프로토콜)
특징
1. 연결지향(3way - handshake) 
2. 데이터 전달 보증
3. 순서 보장

인터넷 프로토콜의 4계층
1. 애플리케이션
2. 전송계층
3. 인터넷 계층
4. 네트워크 인터페이스 계층 (LAN 장비, 드라이버)

애플리케이션 레이어에서 생성된 데이터는 각 레이어를 거치면서
관련 정보를 씌움. 최종적으로 물리계층을 거쳐 인터넷으로 넘어감

생성된 IP패킷은 전송 레이어에서 TCP 세그먼트가 붙게 된다. (출발포트, 도착포트, 순서 등이 명시됨)

UDP
- 데이터 전달 및 순서가 보장되지 않지만 단순하고 빠름
- IP와 거의 같다. 다만 port가 추가


PORT
- 하나의 IP에 둘 이상을 연결 하려면?
- 같은 IP내에서 프로세스 구분
ex) A 아파트(IP) 100동 101호(Port)

DNS
- IP는 기억하기 어렵다
- IP에 대한 도메인명을 저장해둠
- 도메인 명으로 접근한 요청에 대하여 IP를 제공해줌.
